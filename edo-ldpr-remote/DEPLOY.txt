ЭДО ЛДПР — общая база на Render
================================

Идея: данные живут в PostgreSQL на Render. Веб-сервис Flask обрабатывает запросы.
EXE на любом ПК только открывает адрес https://ваш-сервис.onrender.com (или локальный Flask + SQLite, если адрес не задан).

1) PostgreSQL на Render
   - New > PostgreSQL, сохраните имя инстанса.

2) Web Service
   - New > Web Service, подключите этот репозиторий/папку.
   - Build: pip install -r requirements-server.txt
   - Start: см. Procfile (gunicorn edo_ldpr:app_flask)
   - Environment:
     - DATABASE_URL — добавьте «Link» к созданной БД (Render подставит URL).
     - SECRET_KEY — случайная строка (иначе сессии сбросятся при каждом деплое).
   - Платформа Render выставляет RENDER=true — по нему при старте создаются таблицы и сид.

3) Статика
   - Положите static/bg.png рядом с кодом в репозитории (или уберите фон в шаблоне).

4) Клиент (EXE)
   - Рядом с exe создайте файл edo_remote.url с одной строкой: https://ваш-сервис.onrender.com
   - Либо задайте переменную окружения EDO_REMOTE_URL с тем же URL.
   - Если edo_remote.url нет — приложение поднимет локальный Flask и SQLite (режим «один компьютер»).

5) Ограничения Render Free
   - Сервис «засыпает» без трафика; первый запрос может быть долгим.
   - Для продакшена обычно нужен платный план или другой хостинг.
